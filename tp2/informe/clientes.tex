\section{Gestión de clientes}
Este componente es el responsable de la gestión de los datos de clientes. Además
de las funcionalidades básicas de ABM, este componente permite acciones tales
como validar clientes para realizar las operaciones que requieren de autenticación.

Como se describiera en \ref{metodosEstaticos}, la clase Cliente tiene métodos 
estáticos así como métodos convencionales que permiten llevar a cabo las tareas
de ABM. Sin embargo, como también es necesaria la funcionalidad adicional de validación
de usuarios, en este caso se implementa un objeto Controlador que es responsable de
llevar a cabo estas tareas. Esto se hace para respetar SRP.

Las implementaciones de los métodos de estas clases son relativamente sencillas.
El Controlador realiza búsquedas en el conjunto de instancias para chequear los
datos requeridos por la interfaz, y responde en función de los mismos. En principio
no parecen ser necesarios algoritmos sofisticados de búsqueda, pero eventualmente
se podría modificar la implementación de dichos métodos manteniendo su interfaz
para mejorar la eficiencia si se lo juzgara necesario. La encapsulación que provee
el objeto hace que estos cambios no afecten al resto del sistema puesto que tanto
la signatura como la semántica de los métodos se mantendrá intacta.

\subsection{Modelado de escenarios}
\subsubsection{Validación de un cliente}
Cuando se desea ingresar un pedido remoto es necesario ingresar el cliente (esto es opcional si el pedido es local). Para hacer esto, se escriben los datos del cliente, y se realiza la validación. Si es correcta se devuelve al cliente, sino se devuelve NULL.

Por ejemplo consideremos el caso en el que se quiere validar un usuario por su nombre de usuario web y que el cliente existe. También por simplicidad modelaremos el caso en el que el segundo cliente que toma en la busqueda es el adecuado. En general hay que hacer un loop hasta encontrar un cliente, pero para no agregar ruido consideraremos ese caso
